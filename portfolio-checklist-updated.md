## PHASE 1: Technical Foundation

### üî¥ 1.1 Server-Side Rendering / Indexability

**Your Stack:** React 19 + Vite + GitHub Pages + react-router-dom + 404.html fallback

**The Problem:** This is a client-side SPA. When Google's crawler hits `/articles/AWS-Lambda-Practices`, GitHub Pages returns a 404, which triggers your fallback hack to load the React app. But crawlers don't execute JavaScript ‚Äî they see either a 404 or an empty shell.

**Pass Criteria:**
- [ ] `curl https://collinwilkins.com/articles/AWS-Lambda-Practices` returns actual article HTML
- [ ] Google Search Console shows pages as "Indexed"
- [ ] "View Page Source" shows article content (not just `<div id="root"></div>`)

**Your Status:** ‚úÖ **PASS** ‚Äî Migrated to Astro (Static Site Generation)

**Fix Options (pick one):**

| Option | Effort | Result |
|--------|--------|--------|
| **1. Migrate to Astro** | Medium | Static HTML + React islands. Best for content sites. |
| **2. Migrate to Next.js (static export)** | Medium | SSG with `output: 'export'`, still deployable to GitHub Pages |
| **3. Add prerendering to Vite** | Low-Medium | Use `vite-plugin-ssr` or `react-snap` to pre-render routes |
| **4. Switch to Cloudflare Pages + prerender** | Low | Better than GH Pages, some prerender support |
| **5. Use Netlify/Vercel with ISR** | Low | Built-in prerendering, free tier available |

**Recommended:** Option 1 (Astro) or Option 2 (Next.js static export). Both give you:
- Pre-rendered HTML that crawlers can read
- Still deployable to GitHub Pages or Vercel/Netlify free tier
- Better SEO out of the box

**Quick Validation:** Run Google's Rich Results Test: https://search.google.com/test/rich-results
- Enter: `https://collinwilkins.com/articles/AWS-Lambda-Practices`
- If it shows your content ‚Üí you're good
- If it shows empty/error ‚Üí confirms the problem

### üü° 1.4 Meta Tags (Per Page)

**Pass Criteria (each page):**
- [ ] Unique `<title>` (50-60 chars)
- [ ] Unique `<meta name="description">` (150-160 chars)
- [ ] Open Graph tags (`og:title`, `og:description`, `og:image`)
- [ ] Canonical URL

**Your Status:** ‚ö†Ô∏è **LIKELY PARTIAL** ‚Äî SPAs often have static meta tags that don't change per route

**Fix:** Use `react-helmet-async` to set dynamic meta tags per page:

```jsx
import { Helmet } from 'react-helmet-async';

function ArticlePage({ article }) {
  return (
    <>
      <Helmet>
        <title>{article.title} | Collin Wilkins</title>
        <meta name="description" content={article.excerpt} />
        <meta property="og:title" content={article.title} />
        <meta property="og:description" content={article.excerpt} />
        <link rel="canonical" href={`https://collinwilkins.com/articles/${article.slug}`} />
      </Helmet>
      {/* Article content */}
    </>
  );
}
```

**Note:** This only helps if you also fix SSR (1.1). Dynamic meta tags in a client-side SPA are invisible to crawlers.
